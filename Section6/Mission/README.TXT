Réalisé par Martin Gyselinck et Vithit Guilmain

Le code est souple, c'est-à-dire que l'on peut ajouter une commande à l'assistant
simplement en ajoutant un mot dans la liste des commandes (ou alors il y a très peu à ajouter), et en créer sa fonction.
Une fois la base mise en place, il fut donc aisé de réaliser les différentes fonctions.

La principale difficulté rencontrée fut la création du test.
On ne savait pas réellement quoi mettre dedans
Pour fonctionner, le code a besoin d'avoir accès à 2 fichiers :
    - error1.dat
    - error2.dat

Ils étaient proposés sur la page de la phase de réalisation.
Point positif : l'assistant se lance quand le fichier est run, mais si le fichier est importé comme module
(comme dans test.py), alors il ne se lance pas. L'utilisateur a donc la facilité en main.
